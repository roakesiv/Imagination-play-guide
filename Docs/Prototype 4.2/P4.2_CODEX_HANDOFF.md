# P4.2 Codex Handoff - Activity Page Outputs

Use this only after prompt mockup discovery has validated a focused activity-page prompt structure.

---

## Standard Handoff Pattern

```text
I want to continue P4.2 for Magic Creature Card Maker.

Read the top-level source-of-truth docs first:
- 00_README.md
- 01_03_PRODUCT_ROADMAP.md
- 02_APP_SPEC.md
- 03_UX_SPEC.md
- 05_ARCHITECTURE_SPEC.md

Then review:
- Docs/Prototype 4.2/P4.2_SCOPE.md
- Docs/Prototype 4.2/P4.2_PROMPT_DISCOVERY.md
- Docs/Prototype 4.2/P4.2_BUILD_LOG.md
- Docs/Prototype 4.2/P4.2_IMPLEMENTATION_REVIEW.md

Goal:
[Implement one validated activity-page output only]

Validated prompt template:
[Paste validated prompt from P4.2 prompt discovery]

Implementation constraints:
- Use existing P4.0 architecture.
- Put prompt template in promptTemplates.js.
- Use existing creature data contract.
- Use shared prompt builder fallback behavior.
- Add only the intended bridge output/card.
- Do not add any output types beyond the work package.
- Do not change existing templates unless explicitly required.
- Do not add bridge grouping or page split.
- Do not add backend, storage, package dependencies, build step, or APIs.

Documentation:
- Update P4.2_PROMPT_DISCOVERY.md.
- Update P4.2_BUILD_LOG.md.
- Update P4.2_IMPLEMENTATION_REVIEW.md.
- Update P4.2_SCOPE.md status.
- Update top-level docs only for durable changes.

Verification:
- Run manual browser smoke test.
- Confirm new output appears.
- Confirm output uses Fill Example data.
- Confirm Copy bridge output works.
- Confirm existing Coloring and Find-It outputs still work.
- Confirm recent activity outputs still work when relevant.
- Stop and summarize files changed, verification, and risks.
```
