## Recommended Process

1. Use ChatGPT to mock up one activity-page prompt.
2. Test prompt versions manually.
3. Choose the validated prompt structure.
4. Turn the validated template into one focused Codex work package.
5. Implement only that output.
6. Use manual browser smoke testing for implementation verification.
7. Repeat while the activity-page pattern remains lightweight.
8. Use lightweight manual browser review when needed.
9. Close P4.2 when the current activity-page expansion is ready for kid testing.

---

## Important Constraint

Do not add activity types beyond the validated P4.2 activity-page work packages without a focused prompt mockup and implementation pass.
