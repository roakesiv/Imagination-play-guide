## WP1 Image Workspace + Folder-to-PDF Helper Prompt

```text
I want to implement P4.5 WP1 for Magic Creature Card Maker.

Read top-level source-of-truth docs first:
- 00_README.md
- 01_01_PRODUCT_VISION.md
- 01_02_PRODUCT_INSIGHTS_DESIGN_SESSION.md
- 01_03_PRODUCT_ROADMAP.md
- 02_APP_SPEC.md
- 03_UX_SPEC.md
- 04_PROTOTYPE_STARTER.md
- 05_ARCHITECTURE_SPEC.md

Then review:
- Docs/Prototype 4.5/P4.5_README.md
- Docs/Prototype 4.5/P4.5_SCOPE.md
- Docs/Prototype 4.5/P4.5_DISCOVERY_VALIDATION.md
- Docs/Prototype 4.5/P4.5_BUILD_LOG.md
- Docs/Prototype 4.5/P4.5_IMPLEMENTATION_REVIEW.md

Context:
P4.4 reduced prompt-copy friction with the Activity Book Packet. The next blocker is local desktop production after ChatGPT generates the images. Parents currently download many image files, organize them manually, then open and print each image one by one.

Goal:
Create a local desktop helper script that converts a folder of downloaded activity-page images into one multipage printable PDF.

Potential script:
- tools/make_activity_book_pdf.py

Important framing:
- This is a local helper tool, not a web-app feature.
- Generated images and PDFs should live outside the repo.
- Recommended sibling workspace: Imagination-play-images/

Requirements:
- Accept an input folder path.
- Accept optional output PDF path.
- Include PNG, JPG, JPEG, and WEBP files if supported.
- Ignore non-image files.
- Sort files alphabetically by filename by default.
- Include optional sort mode:
  - --sort name
  - --sort modified
- Create one PDF page per image.
- Use US Letter page size by default.
- Fit each image to page while preserving aspect ratio.
- Center image on white background.
- Save a single PDF.
- Print concise summary:
  - number of images included
  - output PDF path
  - sort mode used
- Handle common errors clearly:
  - folder not found
  - no images found
  - image cannot be opened
  - output path not writable

Potential dependency:
- Pillow is acceptable if documented clearly.

Constraints:
- Do not change web app runtime behavior.
- Do not add image upload UI to the web app.
- Do not add direct printing.
- Do not add browser-side PDF generation.
- Do not add backend, accounts, auth, database, package dependencies for the web app, build step, direct image generation, or APIs.
- Do not add generated images or PDFs to the repo.
- Keep this as a local helper tool unless future testing justifies web-app integration.

Docs:
- Update Docs/Prototype 4.5/P4.5_BUILD_LOG.md.
- Update Docs/Prototype 4.5/P4.5_IMPLEMENTATION_REVIEW.md.
- Update Docs/Prototype 4.5/P4.5_DISCOVERY_VALIDATION.md if validation happens.
- Update top-level docs only if a durable decision changes.

Verification:
- Static review of script behavior and docs.
- Run the helper on a temporary local test folder with a small set of non-generated sample images if available.
- Do not use generated family images or commit generated images/PDFs.
- Confirm no runtime app files were changed.
```
