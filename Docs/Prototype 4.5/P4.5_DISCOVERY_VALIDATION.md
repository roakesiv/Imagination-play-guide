## 1. Purpose

```text
Batch prompting works. Printing/local file production is now the blocker.
```

---

## 2. Current Real Workflow

Current production path:

1. Parent generates 10+ activity-page images from ChatGPT.
2. Images download with poor ChatGPT filenames.
3. Parent puts images into a folder.
4. Parent opens each file.
5. Parent configures print options.
6. Parent prints.
7. Parent waits.
8. Parent repeats for each image.

Current pain:

- Too many small manual loops.
- Hard to maintain energy after generating pages.
- Print setup repeats too many times.
- File names do not clearly map to page titles.
- Mistakes are easy when many pages or creatures are involved.

---

## 3. P4.5 Hypothesis

The smallest useful next improvement is a local helper that converts a folder of downloaded images into one multipage printable PDF.

Expected benefit:

```text
print once instead of opening and printing many separate image files
```

## 5. WP1 Validation Plan

WP1 proposed helper:

```text
tools/make_activity_book_pdf.py
```

Validation scenario:

1. Put a small set of generated activity-page images in a local folder outside the repo.
2. Run the helper against that folder.
3. Confirm one PDF is created.
4. Open the PDF locally.
5. Confirm each image appears on its own US Letter page.
6. Confirm images preserve aspect ratio and are centered on white.
7. Print the PDF or review print preview.
8. Compare effort against opening/printing individual PNG files.

Success signal:

- The parent can produce one printable PDF with much less effort than printing separate files.

Failure signal:

- Dependency setup is too hard.
- Image ordering is confusing.
- PDF layout is poor.
- Parent still needs too much manual organization before the script is useful.
