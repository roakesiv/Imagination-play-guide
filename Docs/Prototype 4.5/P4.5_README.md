## Purpose

P4.5 follows P4.4 batch production workflow work.

P4.4 confirmed that the Activity Book Packet reduced repeated prompt-copy friction. The parent can copy one packet prompt into ChatGPT, generate one page at a time, and use `NEXT` to continue.

The next blocker happens after images are generated:

```text
download images -> organize files -> open each image -> configure print -> print -> repeat
```

P4.5 explores a local desktop production pipeline for turning downloaded activity-page images into organized printable files with much less manual work.

This is not a web-app feature yet. It can start as a local helper tool because the images already live on the parent's laptop.

---

## Product Framing

The immediate win is not adding more activity pages or changing the web app.

The immediate win is:

```text
folder of PNG/JPG activity pages -> one multipage printable PDF
```

That would reduce the current "print 10 images one by one" bottleneck.

## Recommended Local Workspace

Generated images and PDFs should not be committed to the GitHub repo.

Recommended sibling-folder setup:

```text
AI Hub/
  Op Revan/
    imagination-play-app-repo/
      Imagination-play-guide/      repo
      Imagination-play-images/     local-only image workspace
```

Recommended `Imagination-play-images` structure:

```text
Imagination-play-images/
  _incoming/
  _rename_manifests/
  pdf-output/
  creatures/
  README.md
```

The exact structure can change during P4.5 discovery, but the principle should not: keep generated images and PDFs outside the repo.
