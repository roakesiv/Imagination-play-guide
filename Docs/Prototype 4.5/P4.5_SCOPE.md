# P4.5 Scope - Local Production Pipeline

**Project:** Magic Creature Card Maker / Imagination Play Guide
**Prototype:** P4.5
**Prototype type:** Local desktop production workflow
**Status:** WP1 implemented / ready for review

---

## 1. Goal

Create a local production pipeline that helps the parent turn downloaded activity-page images into organized printable files with much less manual work.

The first target is the current print bottleneck:

```text
print 10+ images one by one
```

P4.5 should test whether a small local helper can turn a folder of downloaded activity-page images into one multipage printable PDF.

---

## 2. Why This Follows P4.4

P4.4 solved the first major production bottleneck:

- Activity Book Packet reduced repeated prompt copy/paste.
- `NEXT` worked for one-page-at-a-time generation.
- Page titles/names improved the batch flow.
- Short visible instructions near the Activity Book Packet button helped users understand the flow without reading the long generated prompt.

P4.4 did not solve the later desktop production burden:

- generated images download with poor ChatGPT filenames
- parent manually moves images into folders
- parent opens each file
- parent configures print options repeatedly
- parent prints and waits one page at a time

P4.5 starts there.

---

## 3. Learning Question

> Can a local desktop helper reduce the admin burden of turning downloaded activity-page images into a printable activity book?

Secondary questions:

- Is folder-to-PDF the smallest useful first win?
- Does alphabetical filename sorting work well enough for WP1?
- Does modified-time sorting help when ChatGPT filenames are bad?
- Is Pillow an acceptable local dependency for this project?
- Should rename-from-manifest come before or after PDF generation?
- What local workspace structure is easiest for a tired parent to use?

---

## 4. Recommended Local Image Workspace

Generated images and generated PDFs should stay outside the GitHub repo.

Recommended sibling folders:

```text
Imagination-play-guide/       repo
Imagination-play-images/      local-only image workspace
```

Recommended local-only workspace:

```text
Imagination-play-images/
  _incoming/
  _rename_manifests/
  pdf-output/
  creatures/
  README.md
```

Potential meaning:

- `_incoming/` - raw downloads from ChatGPT
- `_rename_manifests/` - future manifest files for renaming downloaded images
- `pdf-output/` - generated PDFs
- `creatures/` - optional per-creature organization later

This folder should not be added to the repo or committed.

## 5. Work Packages

### WP1 - Image Workspace + Folder-to-PDF Helper

Status: implemented / ready for review.

Goal:
Create a desktop helper script that converts a folder of downloaded activity-page images into one multipage PDF.

Scripts:

```text
tools/init_image_workspace.py
tools/make_activity_book_pdf.py
```

Expected requirements:

- Accept an input folder path.
- Accept optional output PDF path.
- Include PNG, JPG, JPEG, and WEBP files if supported.
- Ignore non-image files.
- Sort files alphabetically by filename by default.
- Include optional sort mode:
  - `--sort name`
  - `--sort modified`
- Create one PDF page per image.
- Use US Letter page size by default.
- Fit each image to page while preserving aspect ratio.
- Center image on white background.
- Save a single PDF.
- Print concise summary:
  - number of images included
  - output PDF path
  - sort mode used
- Handle common errors clearly:
  - folder not found
  - no images found
  - image cannot be opened
  - output path not writable

Potential dependency:

- Pillow is acceptable if documented clearly.

## 7. Out Of Scope For P4.5

- Web app runtime behavior changes unless separately approved.
- Image upload UI in the web app.
- Browser-side PDF generation.
- Direct printing.
- Direct AI/image generation.
- Backend services.
- Accounts, auth, database, cloud sync, or APIs.
- Package dependencies for the web app.
- Build step for the web app.
- New activity page types.
- Committing generated images or PDFs.
