## WP1 — Architecture Docs and Current Code Assessment

### Status
Complete / ready for review.

### Files Reviewed

- `Docs/Prototype 4.0/P4.0_BUILD_LOG.md`
- `Docs/Prototype 4.0/P4.0_IMPLEMENTATION_REVIEW.md`
- `Docs/Prototype 4.0/05_ARCHITECTURE_SPEC_DRAFT.md`

### Current Architecture Summary
- `index.html` owns the static app shell and loads `content.js` then `script.js`.
- `content.js` already cleanly owns editable UI content: fields, parent tips, style chips, and example values.
- `script.js` currently owns too many responsibilities: form rendering, DOM reads, creature data creation, main image prompt text, bridge output templates, Find-It item generation, summary/details rendering, copy behavior, reset behavior, and selected output state.
- `styles.css` owns visual layout and P3.2 UX safety placement for Reset.

### Current Prompt / Output Locations
- Main image prompt lives in `script.js` as `buildPrompt(values)`.
- Bridge output templates live in `script.js` as `artifactTemplates`.
- Missing-field fallback lives in `script.js` as `promptValue(value)`.
- Find-It derived item generation lives in `script.js` as `getFindItItems(values)`.
- Summary and details rendering also live in `script.js`, but they are UI recap behavior rather than prompt templates.

### Smallest Safe Refactor Proposal
- Add `promptTemplates.js` after `content.js` and before `script.js`.
  - Move `buildPrompt`, `artifactTemplates`, and Find-It template-specific helper behavior into this file.
  - Expose templates through a single global namespace, for example `window.creaturePromptTemplates`.
- Add `promptBuilder.js` after `promptTemplates.js` and before `script.js`.
  - Own shared helpers such as missing-value fallback, attribute rendering, template lookup, and template rendering.
  - Expose helpers through a single global namespace, for example `window.creaturePromptBuilder`.
- Update `script.js`.
  - Rename or replace `getValues()` with explicit `getCreatureData()`.
  - Keep UI rendering, event wiring, summary/details rendering, copy, reset, and selected output state in `script.js`.
  - Call the builder/templates instead of owning prompt text directly.
- Update `index.html`.
  - Load `promptTemplates.js` and `promptBuilder.js` between `content.js` and `script.js`.

### Behavior Preservation Requirements
- Do not change generated prompt wording except where mechanical migration makes whitespace unavoidable.
- Do not add new output types.
- Do not change parent tips, bridge UI, copy behavior, reset behavior, or content structure.
- Preserve missing-field fallback as `not specified`.
- Preserve GitHub Pages/static file compatibility.

### Recommended Next Step
Proceed to WP2 only after review: extract prompt/output templates into `promptTemplates.js` while preserving output text and behavior.
