# P4.0 README — Architecture Runway

**Project:** Magic Creature Card Maker / Imagination Play Guide  
**Prototype:** P4.0  
**Focus:** Architecture runway for prompt-based creative outputs  
**Status:** Complete / ready for V4.1 planning

---

## Purpose

Prototype 4.0 creates an architecture runway for the next product phase.

V3 established the baseline usable creature-creation app:

```text
parent-guided creature creation
→ child-created creature details
→ AI-ready prompt
→ creature summary and detail recap
→ copyable outputs
→ lightweight bridge options
```

V4.0 should refactor the internal structure so Activity Book and Story Guide features can grow without turning the code into tangled prompt logic.

---

## Current Product Direction

V3 is complete.

Current roadmap:

```text
V3 — Complete: baseline creature creation + UX + docs + content editability
V4.0 — Architecture runway
V4.1 — Coloring + Find-It prompt quality
V4.2 — Maze + Letter Tracing
V4.3 — Small Activity Pack
V5.0 — Story / DM Guide
```

---

## V4.0 Goal

Create a modular prompt-output architecture that supports near-term Activity Book and Story Guide features while keeping the app:

- static
- lightweight
- GitHub Pages compatible
- dependency-free
- easy to understand
- easy to test
- easy to extend

## Recommended Workflow

1. Read top-level source-of-truth docs.
2. Read this P4.0 folder.
3. Use `P4.0_CODEX_HANDOFF.md` to start Codex.
4. Ask Codex to implement one architecture work package at a time.
5. Verify behavior did not change from the user perspective.
6. Use top-level `05_ARCHITECTURE_SPEC.md` as the canonical architecture spec.
7. Keep `05_ARCHITECTURE_SPEC_DRAFT.md` as historical P4.0 draft context.
