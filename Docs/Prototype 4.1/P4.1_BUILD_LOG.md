## Lesson Learned - Testing Scope

- P4.1 prompt-template work does not need automated browser testing as a work-package requirement.
- The app is intentionally simple, static, and dependency-free; manual browser smoke testing is a better fit for the current NFRs.
- The DevTools-driven Edge smoke test eventually worked, but it exposed developer-environment friction that is ahead of the current architecture runway.
- Do not expand the dev environment, add packages, or add automated browser tooling only to verify focused prompt-template changes.
- Future work packages should use lightweight manual smoke testing unless a future NFR or architecture work package explicitly introduces a test runner.
- Mobile and play-trial validation should happen after commit/publish as product validation, not as a blocking implementation-test burden for small template WPs.
