# P4.1 Scope - Prompt Quality

**Project:** Magic Creature Card Maker / Imagination Play Guide
**Prototype:** P4.1
**Status:** Complete

---

## Prototype Goal

Improve prompt quality for printable activity outputs through rapid prompt-specific work packages.

## Working Process

P4.1 uses a rapid prompt-improvement loop:

1. Mock up a prompt manually in ChatGPT.
2. Validate that the generated output works.
3. Turn the validated template into one focused work package.
4. Implement that template.
5. Manually smoke test the app in browser when practical.
6. Document the result.
7. Move to the next prompt only after review.

Testing should scale with the current non-functional requirements and architecture runway. For P4.1 prompt-template work, manual browser smoke testing is sufficient because the app is intentionally static, dependency-free, and easy to inspect directly. Do not add automated browser tooling, package dependencies, or developer-environment expansion only to satisfy a prompt-template work package.

## Work Package 1 - Coloring Page Only

WP1 updates only the existing Coloring Page output template in `promptTemplates.js`.

### WP1 In Scope

- Replace the existing Coloring Page prompt body with the validated coloring-page structure.
- Use the existing P4.0 creature data contract.
- Use shared prompt builder fallback behavior for missing fields.
- Use the creature name when available.
- Gracefully use `the magic creature` when name is missing.
- Keep the existing output type id and UI behavior.
- Manually verify Coloring Page output with Fill Example when practical.
- Manually verify at least one other bridge output still works when practical.
- Manually verify Copy bridge output still works when practical.

### WP1 Out Of Scope

- Find-It prompt updates.
- Maze, Letter Tracing, or any new output type.
- Bridge UI changes.
- Creature form changes.
- Image prompt changes.
- Creature Card, Story, Adventure, or Find-It template changes.
- Backend, storage, package dependencies, build step, or API integration.

## Work Package 2 - Find-It Prompt Quality

WP2 updates only the existing Find-It Game output template in `promptTemplates.js`.

### WP2 In Scope

- Replace the existing Find-It Game prompt body with the validated Find-It activity-page structure.
- Use the existing P4.0 creature data contract.
- Use shared prompt builder fallback behavior for missing fields.
- Use the creature name when available.
- Gracefully use `the magic creature` when name is missing.
- Keep the existing `findIt` output type id and UI behavior.
- Keep the bridge UI the same.
- Do not add new form fields.
- Do not add a separate image list UI.
- Manually verify Find-It Game output with Fill Example when practical.
- Manually verify at least one other bridge output still works when practical.
- Manually verify Copy bridge output still works when practical.

### WP2 Out Of Scope

- Coloring Page changes.
- Story, Adventure, Creature Card, or image prompt template changes.
- Maze, Letter Tracing, or any new output type.
- Bridge UI changes.
- Creature form changes.
- Backend, storage, package dependencies, build step, or API integration.

## Acceptance Criteria

- Coloring Page output uses current creature data.
- Coloring Page prompt says black-and-white printable coloring page.
- Coloring Page prompt says young child.
- Coloring Page prompt says bold clean outlines.
- Coloring Page prompt says large simple shapes.
- Coloring Page prompt says open spaces for coloring.
- Coloring Page prompt says minimal detail.
- Coloring Page prompt says no shading or grayscale.
- Coloring Page prompt says no tiny patterns.
- Coloring Page prompt says no heavy textures.
- Coloring Page prompt says no scary details.
- Coloring Page prompt says not an adult coloring page.
- Other bridge outputs still work.
- No runtime behavior changes outside the Coloring Page template.

## WP2 Acceptance Criteria

- Find-It output uses current creature data.
- Find-It prompt says black-and-white printable hidden-object activity page.
- Find-It prompt says young child.
- Find-It prompt requests the creature name clearly at the top.
- Find-It prompt says the scene should be readable and uncluttered.
- Find-It prompt requests hidden objects.
- Find-It prompt requests a bottom row with picture icons and item names.
- Find-It prompt says pre-reading children and children learning to read.
- Find-It prompt says easy to print and easy to color.
- Find-It prompt avoids dense clutter and difficult puzzle style.
- Other bridge outputs still work.
- No runtime behavior changes outside the Find-It Game template.
