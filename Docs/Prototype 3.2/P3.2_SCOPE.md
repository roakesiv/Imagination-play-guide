# P3.2 Scope

## Goal

Improve first-time adult facilitation clarity and maintainability without expanding product scope.

## In Scope

- Clarify that the app creates prompts, not images.
- Move editable content into `content.js`.
- Rewrite field questions, suggestions, placeholders, and examples.
- Differentiate major buttons and bridge choices with distinct colors.
- Improve header/title/intro readability.
- Add compact expandable Parent Tips / How to Use guidance.
- Update P3.2 docs and durable top-level docs.

## Out of Scope

- Direct AI image generation.
- New bridge output types.
- Saved creature library.
- Printing support.
- PDF export.
- Backend, database, accounts, auth, or persistent storage.
- Framework migration.
- Full story guide or parent DM assistant.
- Major visual redesign.

## Acceptance Criteria

- First-time users can tell the app generates prompts, not images.
- Field/question/suggestion content is centralized in `content.js`.
- Field copy is shorter, more natural, and more glanceable.
- Suggestion examples better support absurd/hybrid creation.
- Major buttons are visually distinct and easier to guide by color.
- Header/intro text no longer conflicts with rainbow/background.
- Parent Tips section is visible, expandable, readable, and not intrusive.
- P3.1 creature creation still works.
- P3.1 bridge outputs still work.
- Copy Prompt works.
- Bridge Copy works.
- Reset works.
- No major new feature scope was added.
- P3.2 docs are created/updated.
- Durable requirements/UX rules are promoted to top-level docs.

## Deferred Validation

Mobile validation for header readability, button colors, and Parent Tips should happen after P3.2 is committed and published.
